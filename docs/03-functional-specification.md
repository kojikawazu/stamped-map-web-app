# Functional Specification (機能仕様書)

## 画面一覧

| 画面 | パス | 説明 |
|------|------|------|
| ログイン | `/login` | メール/パスワードでログイン |
| メイン（地図+リスト） | `/` | スポット一覧 + 地図表示（認証必須） |

> サインアップ画面は不要。アカウントは Supabase ダッシュボードで事前作成する。

## 画面構成

### ログイン画面（`/login`）

- メールアドレス・パスワード入力フォーム
- ログインボタン
- 認証エラーメッセージ表示

### メイン画面（`/`）

```
┌──────────────────────────────────────────────────────┐
│ ヘッダー（ロゴ / ログアウト）                           │
├──────────────────┬───────────────────────────────────┤
│ 左パネル          │ 右パネル（地図）                    │
│                  │                                   │
│ [検索ボックス]    │   MapLibre GL JS                  │
│ [ソート選択]      │   マーカー表示（カテゴリ色）         │
│ [カテゴリフィルター]│   クラスタリング                    │
│                  │   クリック → 登録モーダル            │
│ スポット一覧      │   マーカークリック → ポップアップ     │
│  - スポット1      │                                   │
│  - スポット2      │                                   │
│  - ...           │                                   │
│                  │                                   │
│ [ページネーション] │                                   │
├──────────────────┴───────────────────────────────────┤
│ （登録・編集モーダル：地図上に重ねて表示）                │
└──────────────────────────────────────────────────────┘
```

- PC向けレイアウト（モバイル対応は Phase 2）
- 左パネル：検索 + スポットリスト + フィルター + ソート + ページネーション
- 右パネル：地図表示エリア（メイン）
- メイン画面は全て `'use client'`（MapLibre + 認証ガードのためCSR）

## 認証ガード（クライアント側）

```
1. メイン画面（/）をロード
2. Supabase Auth SDK で認証状態をチェック
3. 未認証 → /login にリダイレクト
4. 認証済み → メイン画面を表示
5. API呼び出し時は JWT を Bearer トークンとして送信
```

> Server Components によるサーバー側リダイレクトは行わない。
> メイン画面は地図（MapLibre）が `'use client'` 必須のため、クライアント側ガードで十分。

## ユーザーフロー

### スポット登録フロー

1. 地図上の任意の場所をクリック
2. クリック地点の座標（latitude, longitude）を取得
3. 登録モーダルが表示される
4. 入力項目：
   - 名前（必須）
   - カテゴリ（必須、プルダウン選択 + カスタム追加）
   - 訪問日（必須、デフォルトは今日）
   - メモ（任意、テキストエリア）
5. 保存ボタンクリック → API Routes 経由で DB に書き込み
6. モーダルを閉じる → 地図にマーカー追加 + リストに反映

### スポット詳細・編集フロー

1. マーカーまたはリスト項目をクリック
2. スポット詳細がポップアップ or パネルで表示される
3. 編集ボタンクリック → 編集モーダルが表示
4. 内容を変更して更新ボタンクリック
5. API Routes 経由で DB 更新 → 表示に反映

### スポット削除フロー

1. 詳細表示から削除ボタンをクリック
2. 確認ダイアログ表示（「本当に削除しますか？」）
3. 確認後 → API Routes 経由で DB 削除
4. マーカー・リストから除去、トースト通知で完了表示

### カテゴリ追加フロー

1. スポット登録/編集モーダルのカテゴリ選択で「＋新規カテゴリ」を選択
2. カテゴリ名・色を入力
3. 保存 → categories テーブルに追加
4. 以降のカテゴリ選択プルダウンに反映

## カテゴリ設計

### デフォルトカテゴリ（初期データ）

| カテゴリ | 色 | 用途 |
|----------|-----|------|
| 食事 | #EF4444 (赤) | レストラン、カフェなど |
| 自然 | #22C55E (緑) | 公園、山、海など |
| 観光 | #3B82F6 (青) | 名所、寺社仏閣など |
| ショッピング | #A855F7 (紫) | 店舗、モールなど |
| その他 | #6B7280 (グレー) | 上記に当てはまらないもの |

### カスタムカテゴリ

- ユーザーが自由に追加可能
- カテゴリ名と色を指定（アイコンは使用しない、色で識別）
- 編集・削除も可能（使用中のスポットがある場合は警告）

## 地図表現

### マーカー

- カテゴリの色でマーカーを表示
- マーカークリックでポップアップ（名前・カテゴリ名・訪問日）

### マーカークラスタリング

- MapLibre の GeoJSON source `cluster` オプションで実装
- ズームインで個別マーカーに分離
- クラスター上に件数を表示

### 地図位置の記憶

- 地図操作（パン・ズーム）のたびに localStorage に中心座標・ズームレベルを保存
- 次回アクセス時に復元
- 初回アクセスまたは localStorage なしの場合：日本全体（中心: 約136.5°E, 36.5°N、ズーム: 5程度）

## ソート・フィルター・検索・ページネーション

### 名前検索

- 左パネル上部に検索テキストボックス
- 入力でスポット名を部分一致検索
- 検索結果は一覧とマーカー表示の両方に反映
- クリアボタンで全件表示に戻る

### ソート

- 訪問日順（新しい順 / 古い順）
- 登録日順（新しい順 / 古い順）
- デフォルト：訪問日の新しい順

### カテゴリフィルター

- カテゴリのチェックボックスまたはタグで絞り込み
- 複数選択可
- フィルターは一覧とマーカー表示の両方に反映
- 「すべて」で全件表示

### ページネーション

- 1ページあたりの表示件数：20件
- ページ切り替えUI（前へ / 次へ / ページ番号）
- ページネーションは一覧リストに適用（地図は全マーカー表示のまま）
